# GCD-001 — Worked Example

This file contains the initial hand-worked example for A-DAP NDC analysis.

Status: illustrative example  
Version: v0.1  
Related document: `technical-note-v0.3.1.md`

## Purpose

GCD-001 is used to demonstrate how A-DAP models a decision as a temporal custody graph and estimates the Custody Disjunction Number (NDC) under declared adversary and verifier models.

This example is not yet computed by an implemented min-cut solver. It is a manual cut-enumeration example intended to test the method.

## Decision flow

| Time | Event | Artifact |
|---|---|---|
| t0 | Scope manifest committed | S0 |
| t1 | Input received | I1 |
| t2 | Model generates output | O1 |
| t3 | Runtime emits evidence | RTE1 |
| t4 | A-DAP envelope created | E1 |
| t5 | Envelope hashed and signed | Sig(E1) |
| t6 | Envelope enters Merkle batch | MR1 |
| t7 | Merkle root externally anchored | A1 |
| t8 | Action executed | X1 |
| t9 | Execution evidence recorded | EX1 |

## Custody domains

- D0: Scope commitment domain
- D1: Generator domain
- D2: Runtime evidence domain
- D3: Envelope creation domain
- D4: Batch and sequencing domain
- D5: External anchoring domain
- D6: Execution evidence domain

## Coherence dependencies

Under verifier V3:

- D0 -> D4: scope determines expected sequence and coverage.
- D1 -> D3: altered input/output/model requires envelope alteration.
- D2 -> D3: altered runtime evidence requires envelope consistency.
- D3 -> D4: altered envelope requires batch consistency.
- D4 -> D5: altered batch requires external anchor consistency.
- D3 -> D6: registered decision must correspond to executed action.
- D0 -> D6: covered decisions require execution evidence or justified absence.

These edges are verifier-relative. If the verifier does not check a dependency, the corresponding edge is inactive.

## Weak deployment

Assumptions:

- D1, D2, D3, and D4 collapse into one internal domain C1.
- D0 is not externally committed.
- D5 timestamps whatever package it receives after the internal showcase has been assembled.
- V checks hash and timestamp but does not independently verify runtime or execution evidence.

Quotient graph:

- C1 = {D1, D2, D3, D4}
- D5 = external anchor
- D6 either absent or collapsed with C1
- D0 absent or mutable

Minimum cut:

- Compromise C1.

Result:

NDC = 1

If the decision can be declared outside scope after the fact:

NDC = 0

## Medium deployment

Assumptions:

- Merkle root and external timestamp are created before action.
- Generation, runtime, envelope, and sequencing still collapse into one internal domain C1 before anchoring.
- D5 is external.
- V checks hash, sequence, and timestamp, but not independently custodied generation evidence.

Candidate cuts:

| Cut | Effect |
|---|---|
| {C1} | Allows construction of H' before anchoring |
| {D5} | Would affect external anchor, but unnecessary if H' is prepared before anchoring |
| {C1, D5} | Stronger than needed |

Minimum cut:

- {C1}

Result:

NDC = 1

This demonstrates Temporal Anchoring Insufficiency: external anchoring can preserve a pre-assembled showcase state, but it cannot independently witness the causal origin of that state.

## Strong deployment

Assumptions:

- D0 scope is committed before decisions and independently reviewable.
- D2 runtime evidence is generated under a domain disjoint from D1.
- D3/D4 envelope and sequencing are separated from generation.
- D5 external anchoring occurs before action.
- D6 execution evidence is separately custodied.
- V3 checks scope, runtime, envelope, sequence, anchor, and execution.

Collapse classes:

- C0 = {D0}
- C1 = {D1}
- C2 = {D2}
- C3 = {D3, D4}
- C4 = {D5}
- C5 = {D6}

Relevant cut candidates:

| Candidate cut | Result under V3 |
|---|---|
| {C1} | Fails: runtime evidence C2 and envelope C3 diverge |
| {C2} | Fails: generator C1 and envelope C3 diverge |
| {C3} | Fails: generator C1, runtime C2, and external anchor C4 diverge |
| {C4} | Fails if C4 is already externally anchored and not controllable under A |
| {C5} | Fails unless execution-only attack is in scope |
| {C0} | Succeeds only for scope-removal attack; bounded by Scope Ceiling |
| {C1, C2} | Fails: envelope/sequence C3 diverges |
| {C1, C3} | Fails: runtime evidence C2 diverges |
| {C2, C3} | Fails: generator C1 diverges |
| {C1, C2, C3} | Succeeds for pre-anchor coherent showcase, assuming C0 and C5 remain consistent |

Minimum enumerated content-showcase cut:

- {C1, C2, C3}

Illustrative result:

NDC_generation/envelope ≈ 3 under this manual enumeration

Important limitation:

This is not yet a solver-based formal computation. The result is illustrative and depends on the declared collapse classes, adversary model, and verifier model.

## Effective NDC

The applied decision must still satisfy:

NDC_effective = min(NDC_scope, NDC_generation, NDC_envelope, NDC_execution)

Therefore, if scope or execution evidence has NDC=1, the applied decision’s effective NDC is 1, even if the manually enumerated content-showcase cut is approximately 3.

## Key lesson

External anchoring alone does not raise NDC above 1 when the internal operator can assemble a coherent showcase state before anchoring.

NDC greater than 1 requires custody disjunction at the origin of generation, not merely posterior registration.
